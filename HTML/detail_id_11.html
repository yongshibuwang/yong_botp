git的基本使用

后端 2019-07-11 1

ssh链接不上Linux   出现WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!解决方法
使用命令清除所连接的IP     ssh-keygen -R XX.XX.XX.XX 

1，GIT中本地提交

首先本地文件夹已经含有文件（单个文件不要大于50m，）内打开git base这里输入git init 

然后将.gitnore中的忽略规则按情况清除。

输入git add。

输入git commit -m“备注” 

输入git remote添加origin + GitHub上的地址将本地的仓库关联到Github上。

输入git push -u origin master上传代码到github远程仓库


第一次报请确保您拥有正确的访问权限解决方法

ssh-keygen -t rsa -C“username”（注：username为你git上的用户名）

三次回车，有时候会让你确定，输入y就行，如有看到关键的randomart图像是，代表你的SSH密钥生成成功了。

cat~ / .ssh / id_rsa.pub（进入相关路径，不一定是这个，看他提示的路径）

ssh-rsa AAAAB后边一长串就是你的SSH Key了。整个内容复制下来，然后在GitHub上点击图像下的设置，点击ssh .....（有ssh后面的记不住了），将其黏贴到上面就行;

2，或者使用这个方式提交

$ git config --global user.name“Your Name”
$ git config --global user.email“email@example.com”
查看用户信息
git config --list
提交信息
git add。（注意，后面有个点）表示添加目录下所有文件到缓存库，如果只添加某个文件，只需把。换成你要添加的文件名即可;
git commit -m“添加你的注释，一般是一些更改信息”
上传代码到远程库，上传之前最好先拉动一下，再执行命令：git pull origin master更新代码到本地
接着执行：git push origin master

3，从GitHub的克隆到指定位置

不需要创建对应文件夹

git clone你的GitHub对应项目的地址。

4，基本操作

git reset HEAD回退暂存区里的文件
